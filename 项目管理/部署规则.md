# 部署规则

## 📋 规则说明

### 1. GitHub 推送规则
**死命令：** 所有推送到 GitHub 的操作，**必须经过用户同意**！

| 操作 | 是否需要同意 |
|------|--------------|
| 修复紧急 Bug | 需要 |
| 添加新功能 | 需要 |
| 更新文档 | 需要 |
| 任何推送操作 | **需要** |

### 2. 本地开发规则
- 本地修改不需要同意
- 但修改前必须仔细检查
- 修改后必须测试验证

### 3. 紧急情况处理
- 如果网站完全无法访问，可以先进行紧急修复
- 但修复后必须立即通知用户

### 4. 责任人
- 开发者：AI 助手
- 审批人：项目负责人（用户）
