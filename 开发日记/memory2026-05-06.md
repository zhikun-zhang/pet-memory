## 1. 🐱 项目概述

**产品名称**: 土豆拾光  
**产品定位**: 宠物日常记录管理应用  
**目标用户**: 宠物主人  
**使用场景**: 记录宠物的日常、健康、饮食等信息  
**核心价值**: 帮助宠物主人记录毛孩子的成长点滴

## 2. ✨ 功能特点

### 📝 日记记录
- 支持添加文字描述和图片
- 图片存储采用 IndexedDB，避免 localStorage 存储限制
- 支持多种记录类型一键同步

### 🐾 宠物管理
- 添加宠物基本信息（名字、性别、品种、生日等）
- 自定义宠物头像（支持图片压缩）
- 显示陪伴天数和体重趋势
- 点击卡片打开宠物详情页

### 📊 健康管理
- 💉 疫苗记录：追踪疫苗接种情况
- 💊 驱虫记录：提醒定期驱虫
- 🛁 洗护记录：记录美容护理
- 🏥 病例记录：管理健康状况
- ⚖️ 体重记录：图表展示体重变化趋势（Chart.js）

### 💰 记账功能
- 自动汇总所有记录中的消费金额
- 按月份统计消费明细

### 📱 用户反馈
- 提交反馈（支持图片上传，最多3张）
- 查看反馈记录
- 一键导出反馈数据
- 反馈查看器（可视化管理工具）

## 6. 🎯 核心功能实现

### 宠物管理
- `savePet()` - 保存宠物信息
- `editPet()` - 编辑宠物信息
- `deletePet()` - 删除宠物
- `openPetDetail()` - 打开宠物详情页

### 记录管理
- `saveRecord()` - 保存记录
- `deleteRecord()` - 删除记录
- `syncRecord()` - 同步日记与专门Tab数据

### 图片处理
- `imgSave()` - 保存图片到 IndexedDB
- `imgGet()` - 从 IndexedDB 获取图片
- `compressImage()` - 图片压缩（最大600px）

### 数据存储
- 宠物数据：`localStorage.getItem('pet-records-pets')`
- 记录数据：`localStorage.getItem('pet-records-records')`
- 图片数据：IndexedDB `pet-records-images`

## 12. ⚡ 项目踩坑记录

### 已解决的问题

| 问题描述 | 原因分析 | 解决方案 | 影响模块 |
|----------|----------|----------|----------|
| localStorage 存储图片超限 | localStorage 有大小限制（约5MB） | 使用 IndexedDB 存储图片 | 图片上传、日记、反馈 |
| 模态框点击事件冒泡 | 点击内容触发背景点击 | 添加 `event.stopPropagation()` | 所有模态框 |
| 宠物详情页头像可点击 | 误使用编辑页面类名 | 创建 `pet-detail-avatar` 类 | 宠物详情页 |
| JSON 字符串双引号冲突 | HTML 属性中使用 JSON.stringify | 通过 window 对象存储数据 | 反馈图片预览 |

### 技术难点

| 难点 | 描述 | 解决方案 |
|------|------|----------|
| 图片压缩 | 上传图片太大 | Canvas 压缩到最大600px | 图片上传 |
| 数据同步 | 日记与专门Tab数据同步 | 添加 `syncRecord` 函数 | 记录管理 |
| 跨Tab数据更新 | 修改后其他Tab需刷新 | 使用 `storage` 事件监听 | 数据更新 |
| 日期计算 | 宠物年龄、陪伴天数 | Date 对象差值计算 | 宠物信息展示 |

### 经验总结
1. 大容量数据优先使用 IndexedDB
2. 模态框按钮添加 `event.stopPropagation()`
3. 上传前压缩图片
4. 提取公共函数避免重复代码
5. 关键函数添加 try-catch

## 15. 🚀 部署上线记录（2026-05-07）

### 15.1 部署过程详细记录

#### 阶段一：GitHub Pages 部署（首次尝试）

**步骤**:
1. 创建 GitHub 仓库 `pet-memory`（用户名：zhikun-zhang）
2. 初始化本地 Git 仓库：`git init`
3. 添加远程仓库：`git remote add origin https://github.com/zhikun-zhang/pet-memory.git`
4. **遇到问题**：分支名称不匹配（本地是 `master`，GitHub 默认是 `main`）
5. **解决方案**：`git branch -m master main`
6. 成功推送：`git push -u origin main`
7. 配置 GitHub Pages：Settings → Pages → 选择 main branch 和 / 目录
8. **遇到问题**：国内访问不稳定，经常 404

#### 阶段二：Gitee Pages 尝试（因代理问题失败）

**步骤**:
1. 创建 Gitee 仓库 `pet-memory`（用户名：jztech_2_0）
2. 添加远程仓库：`git remote add gitee https://gitee.com/jztech_2_0/pet-memory.git`
3. **遇到问题**：`fatal: ServicePointManager 不支持具有 socks5 方案的代理`
4. **解决方案**：取消代理设置 `git config --global --unset http.proxy`
5. **仍失败**：网络环境问题导致无法推送

#### 阶段三：Vercel 部署（最终成功）

**步骤**:
1. 访问 Vercel 官网：https://vercel.com/
2. 使用 GitHub 账号登录
3. 选择个人项目计划（Hobby 免费版）
4. 完成手机验证
5. 授权 Vercel 访问 GitHub 仓库
6. 选择 `pet-memory` 仓库导入
7. 配置项目：
   - Project Name: `pet-memory`
   - Application Preset: `Other`
   - Root Directory: `./`
8. 点击 **Deploy** 按钮
9. 等待约 2 分钟构建完成
10. **成功部署**！访问地址：`https://pet-memory.vercel.app`

### 15.2 选择 Vercel 的原因

| 平台 | 国内访问速度 | 部署难度 | 稳定性 | 费用 | 最终选择 |
|------|--------------|----------|--------|------|----------|
| GitHub Pages | 🐢 较慢/不稳定 | ⭐ 简单 | ⚠️ 不稳定 | 免费 | ❌ |
| Gitee Pages | 🚀 快 | ⭐⭐ 中等 | ✅ 稳定 | 免费 | ❌（代理问题） |
